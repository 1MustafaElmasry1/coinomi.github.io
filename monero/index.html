Proposal: Monero Light Client libraries and Indexer development.

The goal is to develop open source implementations of monero transaction signing for other languages like Java and Rust. All current libraries like for example python, nodejs are in reality API consumers of monerod, thus an alternative implementation will help other projects. In addition to the signing of the transactions the development of a fast indexer is needed.

Once the above is in place, Monero will be added to Coinomi Wallet by Coinomi for no charge other than the servers running cost.

Main goals:

  • Transaction Java class, loosely compatible with Bitcoinj that can serialize, deserialize and sign provided a list of outputs/mixin-pubkeys. An extensive unit test suite is required.
  • Indexer server that will consume the monerod API to index transaction histories, hold watch-keys (preferably in a HSM), notify the user, organize co-signing etc. The indexer shall be written in Rust, due to its speed, memory safety and powerful type system.
  • Transaction structs, traits for Rust that can serialize, deserialize and sign provided a list of outputs/mixin-pubkeys. An extensive unit test suite is required.

Stretch goals:

  • Cryptonote ring signatures, EdDSA and transaction support for the CrypTech open source HSM board. An extensive unit test suite is required. With the above the indexer can use a CrypTech HSM to enhance privacy, co-sign transactions etc.

Developers needed:

  • Monero protocol knowledge, ring signatures, transaction serialization format.
  • Java or Rust knowledge
  • Async IO (for indexer)
